# Daily Input Template

每天开始时，我可以直接填这份模板。Codex 需要根据可用时间动态安排 SRS、主线规划、启发带练和收口。

## 今日输入

```text
日期：
今天可用时间：
当前主线进度：
今天推进的项目模块：
今天主攻知识点：
今天代码/板子状态：
今天遇到的卡点：
今天是否有英语安排：
今天数学是否可开副线：
今天精力状态：
今天必须完成的现实约束：
我希望你重点映射到：
```

## Codex 启动输出格式

Codex 应先输出一个短规划，不直接写代码：

```text
今日模式：
SRS 预算：
SRS 抽卡策略：
今日主线相关标签：
主线目标：
最小闭环：
不做什么：
预计收口产物：
```

## 时间动态规则

如果我只给一个总时长：

```text
30 分钟：只做一个小闭环，尽量不记 problems
60 分钟：SRS 3 张以内，主线一个功能点
90 分钟：SRS + 主线 + 简短 log
2 小时：允许一个概念深挖，但必须回到代码验证
3 小时以上：可以拆成 SRS、主线、扩展、收口四段
```

如果我给多个时间段：

```text
早上短时间：SRS + 校准
中午：主线推进或概念补强
晚上：代码验证、收口、notes/git
```

## SRS 后校准问题

每张卡复习后，Codex 不要急着讲答案，先让我回答：

```text
1. 这个问题的核心误区是什么？
2. 它对应当前代码的哪个函数或哪一行？
3. 这里的输入、输出、返回值、副作用是什么？
4. 如果涉及指针：对象、地址、指针变量、解引用分别是什么？
5. 如果涉及文件 I/O：文件名、FILE *、文件位置、缓冲区分别是什么？
```

## 启发带练请求句式

我可以直接说：

```text
今天有 X 分钟，先带着我 SRS 后，进行主线任务规划，后启发带练模式
```

Codex 应自动执行：

```text
检查 git 状态
读取 current/log/problems/主线代码
运行 daily/daily_review.py plan
必要时运行 daily/daily_review.py map 或 --related-tag
抽 1-5 个高价值问题
校准理解
规划今日主线
挖空主逻辑让我写
检查和验证
收口
```

## 偏航提醒规则

Codex 需要在以下情况主动提醒：

- SRS 超过预算还在继续扩展
- 一个概念深挖超过 15-20 分钟但没有回到代码
- 代码写了很久没有编译运行
- 最后 10 分钟还在开新功能
- 我开始追问远超当前阶段的板子、后端、模型，但今天主线还没闭环

提醒方式要温和但明确：

```text
这个问题值得深挖，但它不是今天的阻塞点。我们先把当前函数跑通，收口后把它记入 problems。
```

## 标签驱动复习

如果今天主线卡在具体概念，Codex 需要优先使用 `--related-tag`：

```bash
python3 daily/daily_review.py list --minutes 60 --related-tag "FILE *"
python3 daily/daily_review.py list --minutes 60 --related-tag CSV
python3 daily/daily_review.py list --minutes 60 --related-tag 返回值
```

抽卡后要说明为什么抽这些卡：

```text
这张卡映射到今天的 save_pond_record_csv()
这张卡补 FILE * / 文件入口
这张卡补返回值合同和失败路径
```
