# Skill Template

用于把一类反复出现的问题沉淀成本地 skill。Skill 不是流水账，而是“以后遇到同类问题时可以快速调出来的能力模板”。

## Skill 名称

一句话命名，例如：

```text
C 文件 I/O 保存层最小闭环
```

## 触发场景

什么时候需要调用这个 skill：

- 例：需要把结构体记录保存到 CSV
- 例：需要解释 `FILE *fp`、`fopen`、`fprintf`、`fclose`
- 例：需要区分文件名、文件入口、文件位置和运行产物

## 它解决什么问题

用现实场景描述，不先讲语法：

```text
项目需要把内存中的一条 PondRecord 记录长期保存下来，供后续查询、统计、上传和训练模型使用。
```

## 第一性原理拆解

按数据流写：

```text
内存中的结构体对象
-> C 标准库文件入口
-> 格式化成文本
-> 写入操作系统文件
-> 关闭并刷新
-> 返回成功或失败状态
```

## 关键代码模板

保留挖空版，方便启发带练：

```c
int save_xxx(Record record)
{
    FILE *fp = fopen(FILE_NAME, "a");

    if (__________)
        return 0;

    // 可选：判断空文件并写 header

    fprintf(fp, "__________\n", __________);
    fclose(fp);
    return 1;
}
```

## 常见误区

- 把文件名当成已经打开后的文件入口
- 把 `FILE *fp` 的声明写进函数调用里
- 把失败路径和成功路径写反
- 忘记 `fprintf` 是副作用，会改变外部文件
- 把普通函数成功返回 `1/0` 和 `main return 0` 混为一谈

## 启发式问题

用于带练，不直接给答案：

```text
这个函数的输入是什么？
它的返回值表达什么合同？
它的副作用是什么？
失败时还有没有可操作的文件入口？
这一步需要文件名，还是 FILE *？
如果文件已经有内容，为什么不重复写表头？
```

## 最小练习

每个 skill 必须带一个可运行闭环：

```text
输入一条记录
运行程序
检查终端输出
检查文件内容
再次运行
确认不会重复写 header
```

## 收口标准

- 能用自己的话解释
- 能指出之前误区
- 能在代码里定位
- 能编译运行验证
- 能说清成功路径和失败路径
