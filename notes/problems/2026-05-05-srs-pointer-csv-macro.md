### 问题 1
- `if (save_pond_record_csv(record) == 0)` 中，`save_pond_record_csv(record)` 是不是只要程序走到这一行就一定会执行一次？

### 解答
- 是。只要程序执行到这个表达式，函数调用会先发生。
- 执行顺序是：
- 调用 `save_pond_record_csv(record)`。
- 函数内部执行 `fopen / fprintf / fclose`。
- 函数返回 `1` 或 `0`。
- 返回值再和 `0` 比较。
- 根据比较结果决定是否进入 `if` 代码块。
- 所以这里不是“先判断再决定是否调用函数”，而是“先调用函数，再用返回值判断”。
- 当前项目合同是：保存成功返回 `1`，保存失败返回 `0`，所以 `== 0` 表示保存失败。

### 闭环
- 用一句话复述：
- `函数调用 -> 副作用发生 -> 返回值产生 -> 参与条件判断`。
- 代码上更清晰的写法是：
- `if (!save_pond_record_csv(record))`

### 使用场景和启发思考
- 场景：保存文件、发送串口帧、上传云端、控制水泵。
- 启发：放进 `if` 条件里的函数，如果有副作用，副作用也会真实发生。

### 问题 2
- `(*record).temp` 是如何访问的？它和 `record->temp` 是什么关系？

### 解答
- `record` 是一个结构体指针变量，类型是 `PondRecord *`，里面存着某个 `PondRecord` 对象的地址。
- `*record` 表示沿着这个地址找到结构体对象本体。
- `(*record).temp` 表示先找到结构体对象本体，再访问它的 `temp` 成员。
- `record->temp` 是 `(*record).temp` 的简写。
- 编译器底层会根据 `PondRecord` 的结构体布局，用“结构体起始地址 + 成员偏移”找到 `temp`。

### 闭环
- 口头拆解：
- `record`：结构体对象地址。
- `*record`：结构体对象本体。
- `(*record).temp`：结构体对象里的 `temp` 成员值。
- `&record->temp`：`temp` 成员的地址。

### 使用场景和启发思考
- 场景：`scanf("%f", &record->temp)`、传感器读数写入结构体。
- 启发：`->` 不是新魔法，它只是“解引用 + 成员访问”的语法压缩。

### 问题 3
- `PondRecord *record;` 中的 `record` 可以叫结构体指针变量吗？`&record` 是整个地址还是首地址？`*`、`.`、`&` 到底有什么区别？

### 解答
- `record` 可以叫指针变量，也可以更具体地叫“指向 `PondRecord` 的结构体指针变量”。
- 更精确的说法是：`record` 是变量名，它的类型是 `PondRecord *`。
- `&record` 的含义要看此处 `record` 是什么：
- 如果 `record` 是 `PondRecord record;` 这种结构体对象，那么 `&record` 是结构体对象的地址，也就是这块结构体内存的起始地址。
- 如果 `record` 是 `PondRecord *record;` 这种指针变量，那么 `&record` 是这个指针变量自己的地址，类型是 `PondRecord **`。
- “整个地址”这个说法不够准确。地址通常表示起点，类型告诉编译器从这个起点开始看多大、怎么解释。

### 闭环
- 三个运算符分工：
- `&x`：取地址，问“这个对象在哪里”。
- `*p`：解引用，沿着地址找到对象本体。
- `x.member`：成员访问，从结构体对象本体里取成员。

### 使用场景和启发思考
- 场景：结构体输入封装、函数参数设计、外部对象修改。
- 启发：C 把“对象、地址、通过地址找到对象、访问成员”分得很清楚，这正是它适合系统/嵌入式的原因。

### 问题 4
- `const char *temp_status(...)` 是指向只读 `char` 的指针吗？字符串字面量是什么？这和数组退化成首地址有关吗？

### 解答
- `const char *` 可以理解成“指向只读字符序列起点的指针”。
- `temp_status()` 返回的不是单个 `char`，而是 `"low"`、`"normal"`、`"high"` 这种字符串字面量的起始地址。
- 字符串字面量可以理解成程序里固定保存的一串字符：
- `"low"` 对应 `'l'`、`'o'`、`'w'`、`'\0'`。
- 它通常位于只读区域，不能通过返回的指针去修改，所以返回类型写成 `const char *`。
- 这和数组退化有关：字符串字面量在表达式中通常表现为指向首字符的地址。

### 闭环
- 拆解函数声明：
- `const char *`：返回值类型。
- `temp_status`：函数名。
- `(float temperature)`：参数列表。
- 它返回的是状态文本的入口地址，供终端输出、CSV 保存、告警逻辑复用。

### 使用场景和启发思考
- 场景：状态文本、CSV 字段、告警标签、前端状态显示。
- 启发：返回“结果文本”比直接打印更可复用。

### 问题 5
- `float *`、`int *`、`PondRecord *` 的大小一样吗？`PondRecord **` 又是什么意思？

### 解答
- 在同一台 64 位机器上，`float *`、`int *`、`PondRecord *` 通常大小相同，因为它们都存地址，常见是 8 字节。
- 但它们的类型意义不同：
- `float *` 解引用后按 `float` 解释，`p + 1` 前进 `sizeof(float)`。
- `int *` 解引用后按 `int` 解释，`p + 1` 前进 `sizeof(int)`。
- `PondRecord *` 解引用后按整个结构体解释，`p + 1` 前进 `sizeof(PondRecord)`。
- `PondRecord **` 是“指向 `PondRecord *` 的指针”，也就是指针的指针。
- 它本身大小通常仍是一个指针大小，不等于 `sizeof(PondRecord)`。

### 闭环
- 层级例子：
- `PondRecord record;`
- `PondRecord *p = &record;`
- `PondRecord **pp = &p;`
- `p` 存结构体对象地址。
- `pp` 存指针变量 `p` 自己的地址。

### 使用场景和启发思考
- 场景：函数要修改外部指针变量本身、动态分配结构体、结构体指针数组。
- 启发：`PondRecord *` 用来改结构体对象内容；`PondRecord **` 用来改“指向结构体的指针变量”本身。

### 问题 6
- 为什么 `fopen("CSV_FILE_NAME", "a")` 不会使用宏，而 `fopen(CSV_FILE_NAME, "a")` 才会使用宏？

### 解答
- 宏替换发生在预处理阶段，只替换裸露的宏标识符。
- `CSV_FILE_NAME` 是宏名，会被替换成它定义的内容。
- `"CSV_FILE_NAME"` 是字符串字面量，双引号里面的字符不会再被当作宏名替换。
- 当前正确写法是：
- `#define CSV_FILE_NAME "pond_records.csv"`
- `FILE *fp = fopen(CSV_FILE_NAME, "a");`
- 预处理后等价于：
- `FILE *fp = fopen("pond_records.csv", "a");`

### 闭环
- 已验证：
- 错误写法会生成名为 `CSV_FILE_NAME` 的文件。
- 正确写法会生成 `pond_records.csv`。

### 使用场景和启发思考
- 场景：文件名常量、阈值常量、缓冲区大小、配置项。
- 启发：宏是文本替换；字符串字面量是已经被固定下来的文本数据。
