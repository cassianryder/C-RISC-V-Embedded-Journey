## 1. 每日问题

### 问题 1
- 在内存中数据和信号如何区分？

### 解答
- 从物理层看，机器里先只有电平变化、时钟脉冲、导线上的高低状态，这一层更接近“信号”。
- 从编码层看，一串信号被约定成 `0/1` 比特序列。
- 从语义层看，同样一串比特，在不同上下文里可以被解释成：
  - 整数
  - 字符
  - 地址
  - 指令
  - 浮点数
- 所以“数据”和“信号”不是两种完全不同的物质，而是同一物理载体在不同抽象层的不同叫法：
  - 信号：强调它在电路里如何传播
  - 数据：强调它在系统里如何被解释

### 闭环
- 练习：把“电平 -> 比特 -> 数值/字符/地址”这条链口头讲一遍。
- 项目映射：后面学结构体、文件 I/O、串口、网络时，都要反复用到“同一串 bit pattern 在不同层被不同解释”这个观念。

### 使用场景和启发思考
- 场景：内存布局、串口通信、文件格式、网络报文、寄存器值解释。
- 启发：先分清物理层、编码层、语义层，很多“数据到底是什么”的疑问会一下清楚很多。

### 问题 2
- 在 `char` 中数值 `-1` 与 `255` 的补码关系是什么？

### 解答
- 在 8 位二进制里，`-1` 和 `255` 的位模式都可以是：
  - `11111111`
- 差别不在位模式本身，而在“你用什么方式解释它”：
  - 如果按 `signed char` 解释，它是 `-1`
  - 如果按 `unsigned char` 解释，它是 `255`
- 所以：
  - 位模式可以相同
  - 数值解释可以不同
- `char` 本身是否带符号，在 C 里还是实现相关的，所以 plain `char` 不能先天等同于 `signed char` 或 `unsigned char`。

### 闭环
- 练习：把 `0xFF` 同时按有符号和无符号解释一遍。
- 项目映射：后面只要涉及字节流、补码、串口、文件、网络数据包，就必须分清“位模式”和“解释方式”。

### 使用场景和启发思考
- 场景：字节流、类型转换、补码、嵌入式寄存器读取。
- 启发：计算机里很多“值”不是天然写在 bit 里的，而是解释规则给出来的。

### 问题 3
- 补码为什么要取反加 1，背后的数学结构是什么？

### 解答
- 核心不是“老师规定要这样做”，而是模运算。
- 对于 `n` 位整数，机器里的加减法本质上是在模 `2^n` 的环里做运算。
- 一个数 `x` 的相反数 `-x`，在这个系统里等价于：
  - `2^n - x`
- 而 `2^n - x` 在二进制里就等价于：
  - `~x + 1`
- 所以“取反加 1”不是技巧，而是“在固定 n 位下构造加法逆元”的结果。
- 这也是为什么补码能让：
  - 加法器继续工作
  - 减法退化成加法
  - `0` 只有一种表示

### 闭环
- 练习：用 8 位数手算 `5` 和 `-5` 的补码关系。
- 项目映射：后面学位运算、寄存器、溢出、符号扩展时，这个理解会反复出现。

### 使用场景和启发思考
- 场景：补码、算术逻辑单元、类型转换、移位与溢出。
- 启发：很多底层规则不是“背下来”，而是某种数学结构为了让硬件更统一而长出来的。

### 问题 4
- 在补码没发明之前，计算机如何实现减法？

### 解答
- 早期系统并不是一开始就统一使用补码。
- 历史上出现过的方案包括：
  - 原码（sign-magnitude）
  - 反码（ones' complement）
  - 补码（two's complement）
- 在原码和反码系统里，减法实现会更麻烦，因为：
  - 符号和数值分离得更明显
  - `0` 可能有两种表示
  - 运算逻辑更绕
- 补码最终胜出，是因为它让“减法变成加法”更自然，也更适合电路统一实现。

### 闭环
- 练习：比较原码/反码/补码在 `+0` 和 `-0` 上的差异。
- 项目映射：这有助于理解为什么现代 C 和现代机器默认都建立在补码直觉上。

### 使用场景和启发思考
- 场景：计算机组成原理、补码历史、算术电路设计。
- 启发：一个表示法被采用，往往不是因为“更好背”，而是因为它让硬件实现更统一、更省事。

### 问题 5
- 在电路中如何识别最高位？

### 解答
- 电路不会“自己猜”哪一位是最高位，最高位是设计时规定出来的。
- 对一个固定宽度的寄存器或总线，例如 8 位：
  - 每一根线的位置都是约定好的
  - 权值最大的那根线就是最高位（MSB）
- 如果是有符号整数，在补码解释下：
  - 最高位通常也就是符号位
- 所以“最高位”不是电路自动发现的自然属性，而是：
  - 位宽固定
  - 线路顺序确定
  - 解释规则再赋予它语义

### 闭环
- 练习：画一个 8 位寄存器，标出 bit7 到 bit0，再说清楚哪个是 MSB。
- 项目映射：以后只要碰到位运算、寄存器、协议字节格式，MSB/LSB 都会反复出现。

### 使用场景和启发思考
- 场景：寄存器图、位字段、字节序、协议格式。
- 启发：底层很多“最高位/最低位/符号位”的概念，本质都是“硬件位置 + 解释约定”的组合。
